# 🔍 loss_amount と risk_span の根本的な違い

| 項目   | **loss_amount**                                                 | **risk_span**                               |
| ---- | --------------------------------------------------------------- | ------------------------------------------- |
| 役割   | 期待値計算に使う「1回の取引で許容する損失額」                                         | TRI安定性補正用の「戦略全体で経験し得るブレ幅」                   |
| 主な用途 | `expected_profit = win_prob*win_amount - loss_prob*loss_amount` | `stability_factor = win_amount / risk_span` |
| 定義   | “損切りライン” に基づく固定値（例：3万円）                                         | 実際の値動きや戦略のブレ（例：過去データから算出）                   |
| 種類   | **戦術的（Trade単位）**                                                | **戦略的（Portfolio/戦略単位）**                     |
| 性質   | 投資家が決める（主観＋戦略）                                                  | 市場が決める（客観＋統計）                               |
| 例    | 1トレードで最大3万円失う                                                   | 株価が±10万円揺れた可能性がある                           |
| 比喩   | **消防車の消火範囲**                                                    | **台風の風速レンジ**                                |

---

## 🧩 図解で直感的に理解する

```
      ┌────────────── 相場で実際に揺れる範囲（＝risk_span）
      │      -100k     |         0       |     +120k
      │───────────────┼──────────────────┼─────────────
      │
      │         ←　←　Loss cut here（-30k）= loss_amount
      │                    （あなたが決める損切り）
│────┼────────────────────────────────────────────
      時間（トレードごとの戦術）
```

🟡 **loss_amount**
→ 自分が定めた「このあたりで損切りしますよ」のライン（主観）

🔵 **risk_span**
→ 実際の価格変動が「最悪ここまで荒れたかもしれない」というブレ（客観）

---

# 🎯 なぜ両方必要なのか？

### ① loss_amount → 期待値の核

```text
expected_profit = win_prob * win_amount - loss_prob * loss_amount
```

→ トレード単位で「勝てるか負けるか」を数値化。
→ あなたが設定した損切り戦略そのもの。

---

### ② risk_span → TRIに反映（精神的負荷・安定性）

```text
stability_factor = win_amount / risk_span
```

→ 値動きが激しすぎる（risk_span大きい）＝精神汗だく → TRIを減点
→ 値動きがまとも（risk_span小さい）＝余裕あり → TRI上昇

つまり、

> 📌 **loss_amountは「戦略上の損失」
> risk_spanは「市場上の揺れによる心理・構造的負担」**

---

# 🔧 実際の例：LITE の場合（イメージ）

| 項目            | 金額              | 解釈                  |
| ------------- | --------------- | ------------------- |
| loss_amount   | 30,000円         | 損切りライン（あなたが設定）      |
| win_amount    | 60,000円         | リスクリワード2.0          |
| **risk_span** | 80,000〜100,000円 | 急変時に実際揺れた価格幅（市場の荒さ） |

この場合：

```
stability_factor = 60,000 / 100,000 = 0.60
→ TRI5 が 40% 減点（精神負荷評価）
```

# 🔥 だからこそよくある誤解

> 「loss_amount が 3万円なら、リスクもそのくらいでは？」

→ 違います。

* loss_amount：**自分が耐えるライン**
* risk_span：**マーケットがあなたの気持ちを乱す幅**

📍 例：-3万円で損切りできても
　**その途中で -10万円相当の下落が発生していたら精密検査モノのストレス**
　→ だから TRI5 では評価を下げます。

---

# 📌 最後にどう扱えば良いか

| ケース     | loss_amount | risk_span の扱い      |
| ------- | ----------- | ------------------ |
| デイトレ、短期 | 明確（あなた定義）   | ATRや当日値幅で算出        |
| 数ヶ月保有   | 損切りライン明確    | 標準偏差 or 最大Drawdown |
| 配当株・金   | 損切り曖昧       | TRI評価対象外（※）        |

※前回お話しした通り
**配当株や金など “保有型” は TRI 評価から外すべき**

# 💎最終まとめ

```
loss_amount ＝自分の「ここで決断する」損失ライン
risk_span   ＝市場が実際に暴れる範囲（精神的負荷も含む）
```

→ **TRIは精神負荷を含む「総合効率」評価なので、両方別軸で必要。**
